# Architecture Documents Needed

## 1. Broad Team Document
Audience: managers, partner teams, stakeholders.

Focus:
- Why we need a scalable agent/tool/context platform
- What business problems it solves
- How other teams can participate
- Migration direction

## 2. Dev Team Architecture Document
Audience: engineers.

Focus:
- Interfaces
- Components
- Repo structure
- Agent/tool/data-source registration
- Testing approach
- Migration plan

## 3. Codex / Claude Code Guide
Audience: AI coding tools and developers using them.

Focus:
- Architecture constraints
- Coding standards
- Prompt templates
- PR checklist
- File ownership and boundaries

## Where to Put Them
- Main repo: project-specific architecture and AI instructions
- Shared docs repo: broader platform vision and reusable standards
- Shared prompt repo: reusable AI prompts across teams
