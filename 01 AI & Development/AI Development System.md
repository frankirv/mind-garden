# AI Development System

## Goal
Create a personal and work-ready AI-assisted development system using Codex, Claude Code, local LLMs, GitHub issues, reusable prompts, and clear architecture documents.

## Key Areas
- [[Codex and Claude Code Workflow]]
- [[Local Windows AI Server]]

## Principles
- Use AI for scaffolding, refactoring, test generation, and documentation.
- Keep architecture constraints written down so AI tools follow them.
- Prefer issue-level prompts instead of vague task requests.
- Do not rely on line-by-line review for every AI-generated change; use tests, architecture checks, and focused human review.

## Open Questions
- Should reusable prompts live in the main repo or a shared prompt repo?
- Which tasks should run locally vs in cloud?
- How much code review is enough for AI-generated PRs?
